## How This Works

**As a repository:**
- Tools get added when they're built and working
- Resources get documented when they're ready
- Case studies get written when stories are complete
- Nothing gets forced before it's useful

**For Claude Code:**
- Clean structure for agentic development
- Each section has its own README
- Clear organization for automated workflows
- Easy to navigate and extend

**For sharing:**
- Parents/players can find resources
- Other coaches can use tools
- Clear documentation for everything
- Professional presentation without overselling

## Documentation Standards

**File naming:**
- Descriptive, lowercase with hyphens
- Version numbers when iterating (v1, v2)
- Date format: YYYY-MM-DD when relevant

**README files:**
- Every section has one
- Explains what belongs there
- Lists current contents
- Notes what's coming

**Status tracking:**
- Clear "what's done / what's next"
- Time estimates when relevant
- Decisions documented in HANDOFF.md
